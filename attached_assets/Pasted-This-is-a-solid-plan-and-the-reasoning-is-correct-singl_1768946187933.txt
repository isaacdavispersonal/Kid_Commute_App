## ✅ 1) Yes — unifying on `dvh` is fine (with one small adjustment)

Using `dvh` everywhere is good on modern iOS. The only risk is **older iOS versions** that don’t fully support `dvh`.

So instead of *only* dvh, the safest “single source of truth” is:

```css
html, body, #root {
  min-height: 100vh;   /* fallback */
  min-height: 100svh;  /* stable */
  min-height: 100dvh;  /* dynamic */
  background-color: hsl(var(--background));
}
```

That still “unifies” to dvh for devices that support it, while preventing regressions on older devices.

If you truly only test on modern iPhones, you can keep dvh-only, but for TestFlight drivers, you’ll likely hit mixed iOS versions.

---

## ✅ 2) The architecture is correct *if and only if* these are true

### A) Every page lives inside `<main class="overflow-y-auto">`

The “white bar” often comes back when you have a route/page that does:

* `position: fixed` fullscreen overlay
* or `h-screen` container
* or `min-h-screen` container
  outside your normal layout flow

After this change, **search your codebase for these**:

```bash
grep -R "h-screen\|min-h-screen\|100vh\|position: fixed\|fixed inset-0" -n client/src | head -n 50
```

Fix any “full-screen” layouts to use dvh/svh and the correct background.

### B) You don’t rely on body scrolling anywhere

With `html/body overflow: hidden`, any component that assumed body scroll will feel “stuck” unless it’s inside `<main>`.

If anything feels broken, it’s because that component isn’t inside `<main>` or it’s using `position: fixed` incorrectly.

## ✅ 3) Why the white bar may still appear even after these changes

If the bar is at the very top and looks like a **status bar area**, the remaining cause could be:

* status bar overlay style / translucent effect
* mismatch between status bar and webview background

If that happens, the fix is usually:

* explicitly set the status bar style and background via Capacitor StatusBar plugin
  (or configure in native)

But don’t jump there unless the CSS/background fix fails.
