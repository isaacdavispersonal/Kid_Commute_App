Short answer:

Android: you can publish without Android Studio if you can produce a signed .aab bundle some other way (cloud build).

iOS: you cannot publish without a Mac/Xcode unless you use a cloud Mac build service that builds/signs/uploads for you. Apple still requires Xcode/Transporter or an automated uploader via App Store Connect API (which those services provide).

Your Replit tree shows /ios and /android folders—great! That means Capacitor scaffolded the platform projects. But you still need the “final step” builds:

Android → a signed AAB

iOS → a signed IPA uploaded to TestFlight/App Store Connect

Since you don’t have a Mac, here are your practical options:

The no-Mac plan (recommended)
Option A — Use a cloud build service (easiest)

These services run Xcode/Android tools on their Macs and can also upload to the stores:

Ionic Appflow (great fit because you’re using Capacitor)

Codemagic, Bitrise, or GitHub Actions (macOS runner) + Fastlane

(Any of these work; Appflow is the most “Capacitor-native”.)

What you’ll do (step-by-step)
1) Accounts you still need

Apple Developer Program ($99/yr)

Google Play Console ($25 one-time)

GitHub (to host your repo; cloud builders connect to it)

Firebase (for push—already covered earlier)

2) Connect your repo to the cloud builder

Push your Replit project to GitHub (or let Replit autodeploy to GitHub).

In Appflow/Codemagic/Bitrise:

“Connect new app” → select your GitHub repo.

Choose build type: iOS and Android.

3) Provide store credentials/signing

iOS

In your Apple account, create an App Store Connect API Key (Issuer ID, Key ID, and download the .p8 key).

In the cloud builder: add the App Store Connect API key (for uploading to TestFlight).

Let the service handle signing automatically (it can create certs/profiles for your bundle ID) or upload your own certificates/profiles if you prefer.

Android

Use Play App Signing (recommended).

Generate an upload keystore (many cloud builders will generate one for you; or you can generate locally with keytool and upload it).

Add your Google Play JSON service account (Play Console → API access → create service account → give “Release Manager” role → download JSON) so the service can upload releases.

4) Configure build settings

Select the Capacitor platform build.

Ensure your capacitor.config.ts has the correct appId and production server.url.

Add env vars (FCM keys, API base URL) if the build needs them.

5) Run builds

Click Build iOS → it produces a .ipa and can upload to TestFlight automatically.

Click Build Android → it produces a .aab and can push to an Internal/Closed track on Google Play.

6) Finish store listings

Apple: App Store Connect → fill out listing, privacy answers, screenshots → submit the already-uploaded build for review.

Google: Play Console → store listing, Data Safety form, screenshots → promote the uploaded build from Internal/Closed to Production.

That’s it—no Mac required.

Option B — Rent a Mac just for the upload

MacStadium / MacInCloud: rent a Mac for a few hours/days.

Install Xcode (free) → open /ios → Archive and Upload to TestFlight.

For Android, you can install Android Studio or use Gradle CLI, but you can also skip it if a cloud build gives you a signed AAB.

This is workable but more fiddly than Option A.

What about your current Replit build “packaged individually”?

Replit created the platform folders (/ios, /android) but Replit itself doesn’t include Apple’s Xcode toolchain (Apple only ships Xcode on macOS), so you still need one of:

A Mac (local or rented) to produce the signed IPA and upload via Xcode/Transporter, or

A cloud Mac builder (Appflow/Codemagic/Bitrise/GitHub Actions on macOS) to produce the IPA and upload for you.

For Android, you could build on Linux via Gradle if your environment has the Android SDK/NDK—cloud builders give you that out of the box. Replit typically won’t have the full Android SDK preinstalled, so cloud build is simpler.

Tiny FAQ

Q: Do I still need an Apple Developer account if a cloud builder uploads for me?
Yes. The builder uses your Apple account/keys to sign and upload your app.

Q: Can I do Android only without any of this?
You still need Google Play Console. For the build, a cloud builder is easiest; or set up the Android SDK on a Linux box and run Gradle to produce a signed AAB.

Q: What about updates later?
Same process: push code → cloud build → upload to TestFlight/Play → submit.
