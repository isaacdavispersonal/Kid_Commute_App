Your code is *almost* correct, but it contains a critical bug:

# ❗ **You wrote:**

```ts
const productionUrl = import.meta.env.VITE_API_URL;
```

but on iOS, this code is **inside a block that only runs when `isNative` is true**:

```ts
if (isNative) {
    const productionUrl = import.meta.env.VITE_API_URL;
    ...
}
```

That part is fine.

**BUT the fatal issue is here:**

```ts
if (_apiBaseUrl !== null) return;  // Already initialized
```

Since `_apiBaseUrl` starts as `null`, `initializeConfig()` runs once.

But look at this:

```ts
let _apiBaseUrl: string | null = null;
```

Later, inside the `if (!productionUrl) { ... }` block, when API URL is missing:

```ts
_apiBaseUrl = '';
```

Meaning:

* the app hits the error once
* sets `_apiBaseUrl = ''`
* **never tries to load the correct URL again**, even if you rebuild the app properly
* this freezes the state forever until app reinstall

So the real issue is:

👉 Your app is showing "API URL not configured" because the old failed state persists.
👉 Even after you fixed `.env.production`, `_apiBaseUrl` never reinitializes.

---

# ✅ **Fix #1 (required): allow reinitialization**

Change:

```ts
if (_apiBaseUrl !== null) return;  // Already initialized
```

to:

```ts
// Only skip reinit if we already have a real value
if (_apiBaseUrl) return;
```

This way, `_apiBaseUrl = ''` triggers reinitialization **instead of blocking it**.

💡 `_apiBaseUrl === ""` means initialization failed previously.
So we should NOT skip reinit.

---

Add this debug line:

```ts
console.log("[Config] Raw import.meta.env:", import.meta.env);
console.log("[Config] VITE_API_URL read from env:", import.meta.env.VITE_API_URL);
```

Put these right under:

```ts
if (isNative) {
```

```ts
if (isNative) {
```

This will tell us exactly what Vite injected into the build.

# Fix #3 (optional but recommended): Clean up `_apiBaseUrl` handling

Replace:

```ts
let _apiBaseUrl: string | null = null;
```

with:

```ts
let _apiBaseUrl: string | null = null;  // null = not yet initialized
```

* `_apiBaseUrl = productionUrl;` when success
* `_apiBaseUrl = null;` if you want to allow reattempts
* `_apiBaseUrl = ''` indicates failure

But the most important fix is **allowing reinitialization**.
