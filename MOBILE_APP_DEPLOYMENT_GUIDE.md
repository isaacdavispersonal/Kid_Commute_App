### 3. Mobile-Friendly API ✅ ALREADY EXISTS
**Status:** Current API is mobile-ready

**What's already working:**
- RESTful JSON APIs with proper HTTP status codes
- JWT-based authentication (Replit OIDC)
- Role-based access control (parent/driver/admin)
- WebSocket support for real-time updates
- Geolocation data ingestion (Samsara + GPS pipeline)
- ETA calculation infrastructure
- Student/route/vehicle management

**Key Endpoints for Mobile Apps:**
```
Authentication:
- GET  /api/auth/user - Get current user profile
- POST /api/auth/logout - Logout

Parents:
- GET  /api/parent/students - Get children with live progress/ETA
- GET  /api/routes/:routeId/stops - Get route stops with ETAs
- GET  /api/messages/route/:routeId - Get route messages
- POST /api/messages - Send message to driver

Drivers:
- GET  /api/driver/route/:shiftId - Get shift route context
- POST /api/attendance - Mark student attendance
- POST /api/driver/route-progress/update-stop - Complete stop
- POST /api/incidents - Report incident
- GET  /api/driver/shifts - Get assigned shifts

Real-time:
- WebSocket at wss://[domain]/ws?token=[jwt] - Live updates
```

**Your Action:** Use these endpoints in your mobile app

### 2. Mobile App Development (React + Capacitor)
**Your tech stack** (recommended to match our web app):
```
Web Layer:   React + TypeScript + Vite
Bridge:      Capacitor
Native:      iOS (Swift) + Android (Kotlin)
```

**Project structure:**
```
fleettrack-mobile/
├── src/                  # React web app (can share code with our web app!)
├── ios/                  # Xcode project (generated by Capacitor)
├── android/              # Android Studio project (generated)
├── capacitor.config.ts   # App config (appId, name, version)
└── package.json
```

**Required Capacitor plugins:**
```bash
npm install @capacitor/geolocation
npm install @capacitor/push-notifications
npm install @capacitor/local-notifications
npm install @capacitor/network
npm install @capacitor/status-bar
npm install @capacitor/splash-screen
```

**capacitor.config.ts example:**
```typescript
import { CapacitorConfig } from '@capacitor/cli';

const config: CapacitorConfig = {
  appId: 'com.yourcompany.fleettrack',
  appName: 'FleetTrack',
  webDir: 'dist',
  server: {
    url: 'https://your-production-domain.com',  // Point to FleetTrack backend
    cleartext: false
  },
  plugins: {
    PushNotifications: {
      presentationOptions: ['badge', 'sound', 'alert']
    }
  }
};

export default config;
```

**What to build:**
1. **Parent App Features:**
   - Login screen (OIDC flow)
   - Dashboard showing children
   - Live map with bus location (use our `/api/parent/students` endpoint)
   - ETA countdown
   - Message thread with driver
   - Notification preferences

2. **Driver App Features:**
   - Login screen
   - Route dashboard (use our `/api/driver/route/:shiftId` endpoint)
   - Attendance marking (use our `/api/attendance` endpoint)
   - Stop completion (use our `/api/driver/route-progress/update-stop`)
   - Incident reporting
   - Background location tracking (send to our GPS webhook)

### 3. iOS Specific Requirements
**Permissions (Info.plist):**
```xml
<key>NSLocationWhenInUseUsageDescription</key>
<string>FleetTrack needs your location to show your bus on the map</string>

<key>NSLocationAlwaysAndWhenInUseUsageDescription</key>
<string>FleetTrack drivers need background location to track the bus route</string>

<key>UIBackgroundModes</key>
<array>
  <string>location</string>
  <string>remote-notification</string>
</array>
```

**Signing:**
1. Create App ID in Apple Developer portal
2. Enable capabilities:
   - ✅ Push Notifications
   - ✅ Background Modes (location)
3. Create certificates and provisioning profiles
4. Configure in Xcode

**Build & Submit:**
```bash
npx cap sync ios
npx cap open ios
# Use Xcode to archive and upload
```

### 5. Firebase Setup (Push Notifications)
**Steps:**
1. Create Firebase project at https://console.firebase.google.com
2. Add iOS app (with your bundle ID)
3. Download `GoogleService-Info.plist` → add to Xcode
4. Add Android app (with your applicationId)
5. Download `google-services.json` → add to Android Studio
6. **Upload APNs certificate** to Firebase (iOS)
7. **Share FCM Server Key with us** (for backend integration)

**To get FCM Server Key:**
1. Firebase Console → Project Settings → Cloud Messaging
2. Copy "Server key" (legacy) or generate new Cloud Messaging API key
3. Send to us securely - we'll add it to our backend environment variables

### 8. Production Hosting (Moving from Replit)

**Why move:**
- Replit preview URLs aren't suitable for production
- Need custom domain + guaranteed uptime
- Need scaling for mobile app load

**Recommended Hosting Options:**

**Option A: Railway.app** (Easiest)
```
✅ Postgres included
✅ Redis available
✅ Auto-scaling
✅ Custom domain + SSL
✅ Simple deploy from GitHub
Cost: ~$20-50/month

Steps:
1. Push FleetTrack code to GitHub
2. Connect Railway to repo
3. Add Postgres plugin
4. Set environment variables
5. Deploy
```

**Option B: Fly.io** (Most control)
```
✅ Postgres (via Supabase)
✅ Redis available
✅ Multi-region deployment
✅ Custom domain + SSL
Cost: ~$15-40/month

Steps:
1. Install Fly CLI
2. flyctl launch (in FleetTrack directory)
3. Add Postgres via Supabase
4. Deploy: flyctl deploy
```

**Option C: Render** (Balance)
```
✅ Postgres included
✅ Redis available
✅ Auto-deploy from GitHub
✅ Free SSL
Cost: ~$25-60/month
```

**Required Environment Variables:**
```bash
DATABASE_URL=postgresql://...
SESSION_SECRET=random-secure-string
SAMSARA_API_TOKEN=your-token
SAMSARA_WEBHOOK_SECRET=your-secret
GPS_WEBHOOK_SECRET=random-secure-string
FCM_SERVER_KEY=your-firebase-key  # NEW - we'll use this
NODE_ENV=production
```

**Custom Domain Setup:**
1. Buy domain (e.g., fleettrack.app) from Namecheap/Google Domains
2. Point DNS to hosting provider
3. Enable SSL (automatic with most hosts)
4. Update Capacitor config to use your domain

### 10. CI/CD (Nice to Have)

**Fastlane Setup** (automate builds):
```ruby
# Fastfile
lane :ios_beta do
  increment_build_number
  build_app(scheme: "FleetTrack")
  upload_to_testflight
end

lane :android_beta do
  gradle(task: "bundleRelease")
  upload_to_play_store(track: "beta")
end
```

**GitHub Actions** (auto-deploy):
```yaml
name: Deploy Mobile Apps
on:
  push:
    tags: ['v*']

jobs:
  ios:
    runs-on: macos-latest
    steps:
      - uses: actions/checkout@v3
      - run: npm install
      - run: npx cap sync ios
      - run: fastlane ios_beta

  android:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - run: npm install
      - run: npx cap sync android
      - run: fastlane android_beta
```
